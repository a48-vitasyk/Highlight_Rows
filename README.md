# Highlight Rows

Розширення-помічник для панелі тікетів на базі **ISPmanager / BILLmanager**.
Працює на сторінках панелі (домени налаштовані в `manifest.json`). Доступне як
**попап** і як **бічна панель** (Chrome side panel / Firefox sidebar). Мета —
пришвидшити роботу зміни підтримки: підсвітка важливих рядків черги, нагадування,
шаблони відповідей, переклад і командна синхронізація.

---

## Можливості

### Черга тікетів (підсвітка та сигнали)
- **Заблокований запит** — підсвічує рядок, якщо тікет заблоковано вашим іменем
  (читається з тултіпа `blocked_by`). Після порогу рядок миготить + звук/сповіщення.
- **Правила за тегами** — якщо тема містить тег (напр. `[TAG]`), рядок отримує свій
  колір; опційно звук/сповіщення з власним повтором.
- **Послуга в тікеті** — у відкритому тікеті дописує дані клієнта: статус, ОС,
  вартість, дату закінчення, трафік (xx/xx ТБ), id послуги (вмикається тумблером).
- **Сигнал про відповідь клієнта** — стеження за чергою з DOM: коли у ваших тікетах
  з'являється нове повідомлення клієнта.
- **Клієнт чекає на відповідь** (наполегливий режим, тумблер «Наполягати, поки не
  відповіси») — щойно клієнт відповів, рахується час очікування й створюється тиск,
  поки підтримка не відпише: **повторюваний сигнал**, **лічильник на іконці** (скільки
  клієнтів чекає + найдовше), **таймер «Клієнт чекає MM:SS» біля поля відповіді**
  (червоніє з часом) і **підсвітка рядка** в черзі. Спільний пул через Supabase з
  **ескалацією**: спершу дзвонить виконавцю, через N хв — усій команді. Гасне автоматично,
  щойно ви відпишете. Усі такі тікети також зведені в окремий блок **«Клієнт чекає»**
  на вкладці «Головна» (уся команда, відсортовано за найдовшим очікуванням, клік
  відкриває тікет). Час до першої відповіді по людях видно в **статистиці** (внизу
  «Налаштувань»). Працює, доки відкрита вкладка панелі.
- **Швидкі дії проти невдоволення** — у тікеті, де клієнт чекає, під таймером панель:
  **«Вже дивимось»** і **«Апдейт»** вставляють у поле відповіді заготовку мовою клієнта
  (шаблон зі скороченням `hold`/`upd`, або типовий текст), а **⏰+30/+1г/+2г** ставлять
  нагад на тікет на обіцяний час. Після N хв очікування таймер підказує надіслати апдейт.
- **Без відповіді > N год** — ручний скан черги (кнопка): тікети, де давно не
  відповідала підтримка. Скан стартує одразу й показує **живий прогрес**
  (завантаження черги → перевірка тікетів), а знайдені зʼявляються в списку на ходу.

### HeartBeat (Будильники) — спільні нагадування
- Нагадування за номером тікета й часом; банер на сторінці за 5 хв до часу з
  кнопками **Взяти / Відписав / Заглушити (снуз)** і гарячими клавішами **T / D / S**.
- **Спільні (командні)** будильники через Supabase: ескалація (спершу власнику, через
  N хв — усім), хто взяв / хто закрив, заглушення на день.
- З картки **«Мої тікети»** тікет можна передати в Будильники однією кнопкою.

### Шаблони відповідей (вкладка «Відмашки», спільна бібліотека)
- Спільні для команди шаблони (Supabase), з **категоріями** (групування + пошук) і
  **скороченнями**-тригерами.
- Вставка у полі відповіді кількома способами: **автодоповнення** під час набору
  (↑↓ · Enter/Tab · Esc), **скорочення/назва + Tab**, палітра швидкого пошуку
  **Ctrl+K**, або **розумні підказки** за змістом відкритого тікета.
- **Підстановки**: `{ticket} {ip} {os} {start} {expire} {traffic}`, `{service}` (назва послуги), `{serviceid}` (ID послуги),
  а також `{greeting}` (Доброго ранку/дня/вечора — за часом доби й мовою тікета),
  `{date}`, `{time}`.
- **Мовні версії UA/RU/EN** одного шаблону — підставляється за визначеною мовою тікета.
- **Панель форматування** (Markdown, як рендерить BILLmanager): **B / I / S / код /
  список / цитата**, Ctrl+B / Ctrl+I; спливає при виділенні тексту.
- **Захист від видалення (soft-delete).** Створювати й редагувати можуть усі, але
  «видалення» лише **архівує** шаблон (з підтвердженням) — він ховається зі списку й
  відновлюється з розділу **«Архів»** унизу. Жорстке видалення через RLS заборонено
  всім (лишається тільки в Supabase) — тож накопичену бібліотеку не можна знести
  випадково чи одним кліком.

### Переклад (безключовий Google Translate)
- **Переклад вхідних повідомлень** клієнта на «мою мову» (кнопка 🌐 біля повідомлення;
  опційно авто).
- **Переклад чернетки відповіді** на мову клієнта (кнопка 🌐→ у панелі форматування).
- **Автопереклад шаблонів** UA → RU/EN при редагуванні.

### Звук і сповіщення
- 15 вбудованих звуків + завантаження свого; окремі звуки для HeartBeat і тегів/блок.;
  гучність; кнопки прослуховування та **тест сповіщення**.
- OS-сповіщення з **окремим текстом/емодзі на тип** (⏰ HeartBeat, 🔴 заблокований,
  🏷️ тег, ✉️ відповідь клієнта); режим **«накопичувати»** (одне згруповане «Нові
  події (N)») або **«заміняти»**; клік відкриває тікет.

### Облік
- **Логи** командних дій із будильниками (хто/що/коли) + **статистика зміни** за день;
  пошук і сортування. Розташовані внизу вкладки **«Налаштування»**.
- **Мої тікети** — закладки тікетів для швидкого доступу.

### Інтерфейс
- **Шрифт** — перемикач у **Налаштування → «Інтерфейс»** (унизу вкладки): **Inter**
  (типовий), **IBM Plex Sans**, **Manrope** або **Системний**. Шрифти зашиті локально
  (variable `.woff2`, субсети latin+cyrillic у `fonts/`) — без зовнішніх CDN, тож
  працює в межах CSP Chrome/Firefox. Вибір зберігається в `storage.local` (`hrFont`)
  і застосовується миттєво.

> ℹ️ Розширення **не робить автоматичних запитів** до billmgr API (через серверні
> 301-редіректи, що спричиняли розлогінення). Скани черги — лише за кнопкою; решта
> даних читається з DOM сторінки.

## Як це працює (архітектура)

| Файл | Роль |
|---|---|
| `content.js` | усе на сторінці панелі: підсвітка рядків, банер HeartBeat, поле відповіді (шаблони/автодоповнення/Ctrl+K/форматування/переклад), читання даних послуги з DOM, визначення мови тікета. |
| `background.js` | service worker: OS-сповіщення; **хаб Supabase** (усі мережеві виклики спільної бази — тут, бо MV3 + CSP сторінки); **Realtime** через WebSocket; безключовий переклад (`translate.googleapis.com`). |
| `sb.js` | клієнт Supabase (REST + JWT) без залежностей: будильники, логи, шаблони, категорії; дзеркалить дані у `storage.local`. |
| `popup.html/js/css` | інтерфейс попапа/панелі (вкладки **Головна / Тікети / Відмашки / Налаштування**). |
| `fonts/` | зашиті шрифти інтерфейсу (Inter / IBM Plex Sans / Manrope, variable woff2, субсети latin+cyrillic). |
| `styles.css` | стилі елементів, які інжектуються в сторінку панелі. |

**Потік даних.** `content.js` читає чергу/тікет із DOM і малює підсвітку/банери
локально. Командні дані (будильники, логи, шаблони, категорії) ходять через
`background.js` → `sb.js` → Supabase; зміни миттєво розходяться по вкладках через
Realtime і дзеркаляться в `storage.local`. Сповіщення `content.js` замовляє
повідомленням у `background.js`.

**Налаштування та стан.**
- `chrome.storage.sync` → `settings` (налаштування) і `reminders` (синхронізуються між пристроями).
- `chrome.storage.local` → сесія Supabase, дзеркала (`snippets`, `snippetCats`, `myTickets`),
  стан заглушень, завантажені звуки, вибір шрифту (`hrFont`) і теми (`hrTheme`) тощо.

### Бекенд (Supabase)
- **Postgres + RLS + Auth (Google OAuth) + Realtime.** Вхід лише через Google;
  реєстрацію обмежено корпоративним доменом команди (тригер у БД), і той самий
  домен перевіряється **в самих RLS-політиках** (другий рубіж) — доступ до даних
  має лише команда.
- Таблиці: `reminders`, `reminder_logs` (незмінний аудит; тригер-функція не
  викликна напряму через API), `snippets`, `snippet_categories`,
  `awaiting_reply` (спільний пул «клієнт чекає», один рядок на тікет) і
  `awaiting_logs` (час до відповіді для статистики; пишеться тригером на resolve).
- Схема ведеться міграціями в `supabase/migrations/` — застосовуються **вручну**
  в Supabase → SQL Editor. Секрети (Google Client Secret, токени) — лише в дашборді
  Supabase, **ніколи в репозиторії**.

## Структура репозиторію

| Тека | Призначення |
|---|---|
| `HighlightRows/` | збірка для Chrome (Manifest V3, service worker) |
| `HighlightRows-Firefox/` | збірка для Firefox (`background.scripts`, `browser_specific_settings`) |
| `supabase/migrations/` | SQL-міграції спільної бази (застосовуються вручну) |
| `tools/` | допоміжні скрипти (напр. `gen-sounds.js` — офлайн-генератор звуків) |
| `build.ps1` | пакування збірок у `dist/` |
| `dist/` | артефакти збірки (у git кладеться лише готовий `.xpi`) |

Код `content.js` / `background.js` / `sb.js` / `popup.*` / `styles.css` в обох
збірках ідентичний — відрізняється тільки `manifest.json`.

## Встановлення

**Chrome:** `chrome://extensions` → увімкнути «Режим розробника» →
«Завантажити розпаковане» → вказати теку `HighlightRows`.

**Firefox:**
- _тимчасово (до перезапуску):_ `about:debugging` → This Firefox →
  «Load Temporary Add-on» → вибрати `.xpi` із `dist/` (або `manifest.json` із
  теки `HighlightRows-Firefox`);
- _постійно без підпису:_ лише у Firefox Developer Edition / Nightly з
  `about:config` → `xpinstall.signatures.required = false`;
- _для розповсюдження:_ підписати `.xpi` через
  [addons.mozilla.org](https://addons.mozilla.org/developers/).

## Оновлення (вручну)

Перевірки версії всередині розширення немає — оновлюємось вручну:

1. `git pull` у теці репозиторію (або взяти свіжий архів).
2. `chrome://extensions` → ↻ Reload (Firefox: перевстановити `.xpi`).
3. Перезавантажити відкриті вкладки панелі.

> Якщо застосовано нові міграції — виконати їх у Supabase → SQL Editor.

## Збірка

Із теки репозиторію:

```powershell
powershell -ExecutionPolicy Bypass -File build.ps1
```

Скрипт читає версію з `manifest.json` кожної збірки й створює в `dist/`:

- `HighlightRows-<версія>.zip` — пакет для Chrome Web Store;
- `HighlightRows-Firefox-<версія>.xpi` — пакет для Firefox.

Архіви формуються через `System.IO.Compression` з прямими слешами в шляхах
(`images/icon48.png`) і `manifest.json` у корені — як вимагають Chrome/Firefox
(`Compress-Archive` у Windows PowerShell 5.1 цього не гарантує).

## Випуск нової версії

1. Підняти `version` у `HighlightRows/manifest.json` та
   `HighlightRows-Firefox/manifest.json` (тримати однаковими).
2. Виконати `build.ps1`.
3. Закомітити зміни (готовий `.xpi` з `dist/` теж версіонується).
